# CopyForge — AI 文案/博客生成器

## 项目规划

### 目标
基于 Next.js 构建一个面向 MVP 的 AI 文案生成器，支持免费试用 + Creem 付费解锁次数，部署于 Vercel + Neon。

### 功能范围

| 功能 | 状态 | 说明 |
|------|------|------|
| 关键词生成文章 | ✅ 已完成 | 用户输入主题 → AI 生成英文文章 |
| 模板风格切换 | ✅ 已完成 | Blog / News / E-commerce / Academic |
| 免费试用 2 次 | ✅ 已完成 | 新用户默认 freeCount=2 |
| Creem 付费解锁 | ✅ 已完成 | Webhook 回调增加 paidCount |
| 用户登录 (Auth0/GitHub) | ✅ 已完成 | NextAuth v5 多 Provider 支持 |
| 用户 Dashboard | ✅ 已完成 | 查看用量、购买次数、文章历史 |
| 部署到 Vercel | ✅ 可部署 | 配置 vercel.json + env |
| 绑定域名 lxs.best | ⏳ 待配置 | Cloudflare → Vercel |

### 技术栈
- Next.js 16 (App Router, Turbopack)
- Tailwind CSS v4
- Prisma 6 + Neon PostgreSQL
- NextAuth v5 (Auth.js)
- DeepSeek (deepseek-v4-flash)
- Creem Payments
- 部署于 Vercel

### 数据流
1. 用户登录 → NextAuth 创建/查找 User → 设置 JWT Session
2. 输入关键词 → POST `/api/generate` → 校验次数 → DeepSeek → 保存 Article → 返回内容
3. 余额不足 → 前端提示购买 → Creem Checkout → Webhook → 增加 paidCount
4. Dashboard → GET `/api/user` + `/api/user/articles` → 展示用量和文章

### 支付流程
```
[用户点击购买] → [/api/creem/checkout] → [Creem 支付页] → [支付成功]
                                                              ↓
[用户回到 Dashboard] ← [Webhook /api/creem/webhook] ← [Creem 回调]
```

## 部署指南

### 1. 数据库（Neon）
1. 注册 https://console.neon.tech
2. 创建项目 → 复制连接字符串
3. 设置 `DATABASE_URL` 环境变量
4. 运行 `npx prisma db push`

### 2. OAuth 提供商
**Auth0：**
1. 注册 https://manage.auth0.com
2. Applications → Create Regular Web App
3. 设置回调 URL: `https://lxs.best/api/auth/callback/auth0`
4. 复制 Domain / Client ID / Client Secret

**GitHub（可选）：**
1. Settings → Developer settings → OAuth Apps
2. 设置回调 URL: `https://lxs.best/api/auth/callback/github`
3. 复制 Client ID / Secret

### 3. Creem
1. 注册 https://creem.io
2. 创建 Product（10 credits pack, $4.99）
3. 设置 Webhook URL: `https://lxs.best/api/creem/webhook`
4. 复制 API Key / Product ID / Webhook Secret

### 4. Vercel 部署
1. 推送代码到 GitHub
2. 在 Vercel 导入仓库
3. 添加所有环境变量（见 `.env.local.example`）
4. 部署
5. 设置自定义域名 lxs.best
